# WFM 排班系统员工端小程序

基于 **微信小程序原生 + TypeScript + Supabase API** 构建的员工端应用。  
面向 **员工角色**，用于查看个人排班、发起调班申请、查看公告通知、查看个人信息、导出班表图片等。

本仓库定位为 **微信小程序员工端**，后端统一依赖 **Supabase** 提供认证、数据库、存储、Edge Functions 等能力，数据模型与管理后台保持一致。

---

## 1. 项目简介

WFM 排班系统面向短期、项目制灵活用工场景，适用于会展、剧组、调研、动捕、非侵入式合作等业务类型。

员工端小程序的目标是让员工在微信内完成以下核心操作：

- 登录系统
- 首次登录修改密码
- 查看首页排班概览
- 查看月度 / 周度 / 单日排班
- 导出月视图 / 周视图班表图片
- 发起调班申请
- 查看申请 / 审批详情
- 查看公告通知
- 查看个人信息
- 修改密码与退出登录

> 说明：
>
> - 本端仅面向员工角色
> - 员工只能查看自己的数据
> - 员工默认只能查看 **已发布班表**
> - 数据权限由 **JWT + RLS + 业务角色权限** 共同控制

## 3. 技术选型

### 3.1 前端技术栈

| 分类       | 技术                                          | 说明                                      |
| ---------- | --------------------------------------------- | ----------------------------------------- |
| 运行平台   | 微信小程序原生                                | 员工端主运行环境                          |
| 开发语言   | TypeScript                                    | 提升类型安全，与 `schema.md` 字段保持一致 |
| 页面模板   | WXML                                          | 小程序原生视图模板                        |
| 样式方案   | WXSS                                          | 实现卡片、表单、日历、列表等页面效果      |
| 页面组织   | Page / Component                              | 使用原生页面与组件拆分业务                |
| 路由方式   | 小程序页面栈                                  | 原生导航模型                              |
| 底部导航   | 自定义 TabBar                                 | 首页 / 申请 / 排班 / 公告 / 我的          |
| 数据请求   | `wx.request` 封装                             | 对接 Auth / PostgREST / RPC / Functions   |
| 本地缓存   | `wx.setStorageSync`                           | 缓存 token、用户信息、基础配置            |
| 状态管理   | `App.globalData` + 页面 `data` + service 模块 | 轻量状态管理                              |
| 日期处理   | `dayjs`（推荐）                               | 月历、周历、日期格式化                    |
| 图片导出   | Canvas                                        | 用于生成月视图 / 周视图班表图片           |
| 代码规范   | ESLint + Prettier                             | 统一代码风格                              |
| 调试工具   | 微信开发者工具                                | 编译、预览、真机调试                      |
| 自动化发布 | `miniprogram-ci`（可选）                      | 支持 CI 上传发布                          |

### 3.2 后端技术栈

> 员工端小程序与管理端共用同一套后端能力，统一基于 Supabase 构建。

| 分类         | 技术                    | 说明                                              |
| ------------ | ----------------------- | ------------------------------------------------- |
| BaaS 平台    | Supabase                | 提供 PostgreSQL / Auth / Storage / Edge Functions |
| 数据库       | PostgreSQL              | 核心业务数据存储                                  |
| 后端开发语言 | TypeScript              | 用于 Supabase Edge Functions                      |
| 鉴权         | Supabase Auth           | 登录、会话、身份认证                              |
| 权限隔离     | RLS + 业务角色表        | 数据权限与角色权限共同控制                        |
| 文件存储     | Supabase Storage        | Excel 导入文件、导出文件、错误报告文件            |
| 服务函数     | Supabase Edge Functions | Excel 导入导出、批处理、审批发布等复杂逻辑        |

> 说明：
>
> - 上述后端能力是平台统一能力，不代表员工端本期全部启用
> - 员工端本期主要使用认证、排班查询、公告查询、调班申请等能力
> - 班表图片导出由前端 Canvas 完成，不依赖后端文件导出流程

### 3.3 选型说明

员工端采用 **微信小程序原生 + TypeScript**，主要基于以下考虑：

- 员工主要在微信内使用，原生小程序体验更稳定
- 更适合移动端表单、日历、列表、卡片式页面
- 与微信生态能力结合更自然
- 包体更可控，性能更稳定
- TypeScript 更利于字段约束、接口联调和多人协作

> 员工端请求层推荐采用 **`wx.request` + HTTP API** 模式，而不是在小程序中重度依赖 `supabase-js`，以降低兼容性和包体成本。

## 4. 系统架构

员工端采用 **微信小程序前端 + Supabase API** 的前后端分离方案：

```text
微信小程序员工端
  ├─ Auth 接口
  │   ├─ 登录
  │   ├─ 刷新会话
  │   └─ 修改密码
  ├─ PostgREST 接口
  │   ├─ employee
  │   ├─ schedule
  │   ├─ shift_change_request
  │   ├─ announcement
  │   └─ employee_work_metric
  ├─ RPC 接口
  │   └─ get_current_user_permissions
  ├─ Canvas 导出层
  │   ├─ 月视图班表导出图片
  │   └─ 周视图班表导出图片
  └─ Functions 接口
      └─ 后续复杂流程扩展
```

### 核心原则

- 常规读取优先走 **PostgREST**
- 聚合数据优先走 **RPC**
- 复杂流程优先走 **Functions**
- 班表图片导出优先由 **前端 Canvas 本地生成**
- 所有敏感权限必须依赖后端 **JWT + RLS**
- 前端只负责交互和展示，不作为最终权限控制依据
- 员工端默认只查询本人数据、只显示已发布排班

## 6. 页面功能说明

### 6.1 登录页

功能包括：

- 员工账号密码登录
- 记住我
- 忘记密码入口（按实际后端能力启用）
- 登录成功后初始化会话

说明：

- 若实际 Supabase 认证不直接支持“用户名登录”，可通过账号映射或后端代理函数实现
- 登录成功后应缓存：
  - `access_token`
  - `refresh_token`
  - `expires_at`
  - 当前用户资料

### 6.2 首次登录修改密码

功能包括：

- 首次登录强制修改初始密码
- 新密码规则校验
- 修改成功后进入系统首页

建议规则：

- 至少 8 位
- 包含大写字母
- 包含小写字母
- 包含数字

### 6.3 首页

首页主要展示：

- 当前日期与月历概览
- 今日班次
- 未来若干日班次
- 本月工时概览
- 快捷入口

建议数据来源：

- `employee`
- `schedule`
- `employee_work_metric`

### 6.4 申请中心

当前版本申请中心仅开放：

- 调班申请

说明：

- 原型中的休假申请、加班申请本期不开发
- 当前可基于 `shift_change_request` 实现调班申请提交与记录查看

### 6.5 排班页

排班页是员工端核心页面，当前版本支持：

- 月视图
- 周视图
- 日视图

主要功能包括：

- 当前员工信息卡
- 本月应完成工时
- 截止今日完成工时
- 视图切换（月 / 周 / 日）
- 月视图日历
- 周视图班次列表或周历排布
- 日视图班次详情
- 班次图例
- 班次类型筛选
- 指定月份 / 指定周切换
- 某日排班详情入口

建议获取方式：

- 月视图：按当前月份范围查询 `schedule`
- 周视图：按当前周或指定周范围查询 `schedule`
- 日视图：按指定日期查询 `schedule`
- 工时指标：查询 `employee_work_metric`
- 后续如有性能或聚合展示需求，可补充员工端专用 RPC

### 6.6 班表导出图片

当前版本支持：

- 月视图导出图片
- 周视图导出图片

实现方式：

- 小程序前端使用 **Canvas** 绘制导出内容
- 基于当前已加载的班表数据生成图片
- 生成后可进行：
  - 预览图片
  - 保存到相册
  - 转发前手动分享（按微信能力限制处理）

建议导出内容包含：

- 员工姓名 / 工号
- 导出视图类型（月 / 周）
- 时间范围
- 班次图例
- 班表主体内容
- 生成时间

说明：

- 当前不提供独立后端导出接口
- 当前不支持 Excel / PDF 格式导出
- 当前不支持日视图导出图片
- 图片生成效果可能受机型、像素比、字体加载、长内容布局影响

### 6.7 公告页

公告页主要展示：

- 公告分类
- 公告列表
- 公告详情

当前可直接依赖：

- `announcement`
- `announcement_type` 字典

### 6.8 我的

个人中心主要包括：

- 工号
- 姓名
- 部门
- 联系方式
- 入职时间
- 个人资料展示
- 修改密码
- 退出登录
