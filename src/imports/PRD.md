# 企业排班系统 PRD（最终版）

## 一、系统概述

### 1.1 项目背景

本系统面向短期、项目制灵活用工场景，适用于会展、剧组、调研、动捕、非侵入式行为采集等业务类型。

| 业务类型    | 典型应用             | 特征描述            |
| ------- | ---------------- | --------------- |
| 会展/活动   | 展会搭建、演出保障        | 短期爆发、人员调配频繁     |
| 影视/剧组   | 拍摄统筹、现场执行        | 时间不固定、任务驱动      |
| 调研/普查   | 访问、采集、记录         | 分散执行、排班变化快      |
| 动捕/数据采集 | 数据采集、行为记录        | 设备依赖强、技能要求明确    |
| 非侵入式合作  | 医疗、洗衣、烹饪、维修等动作采集 | 借助真实场景与外部人员完成任务 |

典型痛点：

- 排班依赖人工，效率低、调整慢
- 员工、负责人、调度之间信息不同步
- 用工来源复杂，自招、渠道、推荐并存
- 线下 Excel 月排班矩阵大量使用，系统需兼容既有模板
- 员工技能、级别、效率与工时画像未沉淀
- 排班、任务、设备、人员之间未形成闭环

因此，需要建设一个支持**项目制灵活用工**的轻量级排班系统。

***

### 1.2 灵活用工定位

- 用工周期以几周到 1 个月内项目为主
- 工作安排常见为按小时、按天、按周排班
- 人员来源包括自雇、企业自招、渠道推荐、非侵入式合作
- 支持围绕任务与设备进行灵活调配
- 管理方式为**项目制 + 任务制 + 排班驱动**

核心用工模式：

1. **自建场景模式**：企业自行创建项目、任务、设备与排班
2. **非侵入式模式**：与外部行业合作，在真实工作场景中完成任务

***

### 1.3 项目目标

本期目标：

- 支持基于**项目/任务**驱动的排班管理
- 支持多渠道、多技能属性人员管理
- 支持员工、设备、任务状态统一管控
- 支持员工技能级别与效率系数管理
- 支持员工近 7 日、近 30 日平均工时统计
- 支持月度矩阵式排班
- 支持与现有 Excel 排班模板兼容的导入与导出
- 支持**标准 Excel 月度矩阵模板**导入：\
  **员工 + 日期 + 排班编码**
- 支持班表发布、调班申请与审批
- 支持 500 人以下规模稳定运行

本期不包含：

- 智能排班
- 自动推荐排班方案
- 请假审批模块
- 复杂考勤结算模块
- 薪资结算模块

说明：

> 为兼容 Excel，`事 / 病 / 旷 / 离` 等编码仍保留为排班编码，仅用于导入、导出和统计，不形成独立请假审批流程。

***

### 1.4 系统架构

| 端    | 平台    | 用户角色      |
| ---- | ----- | --------- |
| 管理后台 | Web 端 | 管理员、部门负责人 |
| 管理端  | 微信小程序 | 管理员、部门负责人 |
| 员工端  | 微信小程序 | 员工        |

***

### 1.5 核心业务流程

**字典配置 → 运营规则配置 → 场景创建并配置设备 → 项目创建并关联场景 → 项目下创建任务 → 渠道/员工/技能维护 → 任务绑定设备 → 排班生成（手工/模板/Excel 导入）→ 班表版本发布 → 员工查看班表 → 调班申请/审批 → 工时统计 → 统计分析**

其中，Excel 导入流程细分为：

**选择项目 + 选择月份 + 上传标准月矩阵 Excel → 预校验 → 生成草稿版本 → 人工补充任务/设备（可选）→ 发布**

- 本期仅保留与项目制灵活用工排班直接相关的内容

## 三、核心业务模型

### 3.1 业务关系

系统核心关系如下：

- 一个**场景**下可以有多个**设备**
- 一个**项目**关联一个**场景**
- 一个**项目**下可以有多个**任务**
- 一个**任务**可以绑定多个**设备**
- 一个**设备**只绑定一个**技能**
- 一个**员工**可以拥有多个**技能**
- 员工对某项技能可记录一个当前**技能级别**
- 排班时，员工只有持有设备绑定技能时，才能被安排操作该设备

关系表达：

```text
场景 Scene
  └── 设备 Device

项目 Project
  └── 关联场景 Scene
  └── 任务 Task
        └── 任务设备 TaskDevice

设备 Device
  └── 绑定技能 Skill

员工 Employee
  └── 员工技能 EmployeeSkill
        ├── 技能 Skill
        ├── 技能级别 skill_level
        └── 效率系数 efficiency_coefficient

排班 Schedule
  ├── 员工 Employee
  ├── 项目 Project
  ├── 任务 Task（可为空）
  ├── 设备 Device（可为空）
  └── 排班编码 ScheduleCode
```

***

### 3.2 模型说明

- **场景**：表示一个实际作业地点或采集地点
- **项目**：在某个场景中执行
- **任务**：是项目下具体需要完成的工作项
- **设备**：属于某个场景，并绑定一个可操作技能
- **任务设备关系**：表示一个任务需要使用哪些设备
- **员工技能**：用于校验员工是否具备设备操作资格，并记录该员工在该技能上的级别和效率系数
- **技能级别**：本期固定为三档，不独立建表，直接作为 `employee_skill.skill_level` 字段维护
- **排班记录**：默认以员工为主视角
- **标准 Excel 月矩阵导入**：导入的是“员工 + 日期 + 排班编码”的基础班表，任务和设备信息允许为空

***

### 3.3 本期保留内容

- 场景管理
- 项目管理
- 任务管理

- 任务绑定设备

## 五、场景、项目、任务与设备管理

### 5.1 场景管理

场景为独立主数据，表示一个实际作业场地或采集场地，用于记录名称、地点及该场景下的设备资源。

建议字段：

| 字段   | 类型           | 说明   |
| ---- | ------------ | ---- |
| 场景ID | bigint       | 主键   |
| 场景名称 | varchar(100) | 名称   |
| 场景地点 | varchar(200) | 具体地点 |
| 场景描述 | text         | 说明   |
| 是否启用 | bool         | 状态   |

功能要求：

- 新增/编辑/停用场景
- 查看场景下设备列表
- 按场景查看关联项目
- 支持一个场景维护多个设备

***

### 5.2 设备管理

设备属于场景，每个设备绑定一个技能，用于判定员工是否具备操作资格。

建议字段：

| 字段      | 类型           | 说明                 |
| ------- | ------------ | ------------------ |
| 设备ID    | bigint       | 主键                 |
| 场景ID    | bigint       | 所属场景               |
| 设备编号    | varchar(50)  | 唯一编号               |
| 设备名称    | varchar(100) | 名称                 |
| 技能ID    | bigint       | 设备绑定技能             |
| 状态字典项ID | bigint       | 关联 `device_status` |
| 备注      | text         | 说明                 |

规则：

- 一个设备仅属于一个场景
- 一个设备仅绑定一个技能
- 只有持有该技能的员工才可被安排操作该设备
- 设备停用或状态异常时，不可参与排班

***

### 5.3 项目管理

项目是排班业务一级容器，项目需关联具体场景。

建议字段：

| 字段      | 类型           | 说明                           |
| ------- | ------------ | ---------------------------- |
| 项目ID    | bigint       | 主键                           |
| 项目名称    | varchar(100) | 名称                           |
| 场景ID    | bigint       | 关联场景                         |
| 项目模式    | varchar(30)  | `self_built / non_intrusive` |
| 开始日期    | date         | 起始日期                         |
| 结束日期    | date         | 结束日期                         |
| 负责人ID   | bigint       | 项目负责人                        |
| 状态字典项ID | bigint       | 关联 `project_status`          |
| 备注      | text         | 说明                           |

规则：

- 一个项目当前版本默认关联一个场景
- 项目下任务使用的设备应来自该项目关联场景下的设备

***

### 5.4 任务管理

任务是排班前置对象，必须归属于项目。

建议字段：

| 字段      | 类型           | 说明               |
| ------- | ------------ | ---------------- |
| 任务ID    | bigint       | 主键               |
| 项目ID    | bigint       | 所属项目             |
| 任务名称    | varchar(100) | 名称               |
| 总工时目标   | decimal(8,2) | 目标工时             |
| 单次工时    | decimal(5,2) | 单班次工时            |
| 有效率要求   | decimal(4,2) | 如 0.80           |
| 计划开始日期  | date         | 开始日期             |
| 计划结束日期  | date         | 结束日期             |
| 状态字典项ID | bigint       | 关联 `task_status` |
| 备注      | text         | 说明               |

说明：

- 任务本期不单独维护工种或岗位要求
- 任务的设备操作资格由**任务绑定设备 + 设备绑定技能**共同决定

***

### 5.5 任务与设备绑定

一个任务可绑定多个设备，用于表示该任务由哪些设备共同完成。

建议字段：

| 字段   | 类型           | 说明   |
| ---- | ------------ | ---- |
| 关系ID | bigint       | 主键   |
| 任务ID | bigint       | 所属任务 |
| 设备ID | bigint       | 绑定设备 |
| 排序号  | int          | 可选   |
| 备注   | varchar(255) | 说明   |

规则：

- 一个任务可绑定多个设备
- 任务绑定的设备必须来自项目关联场景下的设备
- 同一设备可在不同时间参与不同任务
- 排班时，如选择了任务，则设备必须来自该任务已绑定的设备集合

***

### 5.6 任务测算与完成口径

本期仅做基础测算，不做智能排班。

测算口径：

- 任务目标工时来自任务定义
- 实际排班工时来自班表计划工时
- 实际有效工时可结合员工技能级别效率系数计算
- **仅已关联任务的作业排班记录参与任务完成率统计**

建议公式：

```text
实际有效工时 = Σ(排班计划工时 × 效率系数快照)
任务完成率 = 实际有效工时 / 任务总工时目标
```

说明：

- 标准 Excel 基础导入若未关联任务，则仅计入项目或员工工时，不进入任务完成报表

- 若选择设备，则设备状态必须允许排班
- 若选择设备，则设备必须属于当前项目关联场景
- 若选择设备，则设备必须已绑定到当前任务
- 若选择设备，则员工必须持有该设备绑定技能
- 默认同一设备同一日期同一班次仅允许一个有效作业排班
- 工时上限与连续工作天数按用工规则校验
- 已发布版本不可直接修改明细，只能通过新版本或调班流程变更
- 本期不做智能推荐，只做规则校验

建议唯一约束：

```text
UNIQUE(version_id, employee_id, schedule_date)
```

## 八、Excel 导入导出

### 8.1 设计原则

本期标准导入模板为**员工月度矩阵排班表**，采用固定结构：

- 行表示员工
- 列表示日期
- 单元格表示当天排班编码
- Excel 主要导入“员工 + 日期 + 排班编码”
- 任务、设备、技能快照不作为该模板的必填导入内容

说明：

- 该模板导入后生成的是**项目级基础排班记录**
- 若后续需要任务、设备信息，可在系统中补录
- 后端按固定结构解析，不依赖单元格颜色、字体、边框等样式信息
- 本期标准模板中，A 列业务含义固定为**员工姓名**

***

### 8.2 Excel 导入要求

- 支持 `.xlsx`
- 默认读取第一个 Sheet
- 支持下载标准模板
- 支持兼容现有线下月度矩阵表
- 支持按项目 + 月份导入
- 导入时必须由用户选择：
  - `project_id`
  - `schedule_month`
- 支持导入前预校验
- 支持错误报告下载
- 导入成功后自动重算统计数据
- 导入成功后默认生成草稿版本
- 不允许直接覆盖已发布版本
- 支持导入模式选择：
  1. 覆盖当前草稿版本
  2. 新建草稿版本

***

### 8.3 Excel 导出要求

- 支持按项目/月度导出
- 保持与线下模板一致的版式
- 导出结构采用标准月矩阵模板
- 第1行为标题
- 第2行为日期
- 第3行为星期
- A列为姓名
- B列起为每日排班编码
- 周末/节假日可按模板颜色高亮展示
- 可按需要附加隐藏技术字段，但默认用户可见版式与标准模板一致
- 右侧统计区本期可不强制输出；如输出，仅用于查看，不作为导入依据

***

### 8.4 标准 Excel 模板结构

标准模板结构如下：

| 区域    | 位置     | 说明                              |
| ----- | ------ | ------------------------------- |
| 标题区   | 第1行    | 合并单元格，示例：`项目排班表(2026年4月)`       |
| 日期区   | 第2行    | 从 `B2` 开始，依次填写 `1 ~ 月末`         |
| 星期区   | 第3行    | 从 `B3` 开始，依次填写对应星期；`A3` 固定为“姓名” |
| 员工列   | A列     | 从 `A4` 开始，每行一个员工姓名              |
| 排班矩阵区 | `B4` 起 | 每个单元格填写一个排班编码，如 `休 / 捕1 / 检2`   |

模板要求：

- 默认读取第一个 Sheet
- 日期列数按自然月天数动态变化（28/29/30/31）
- 单元格颜色仅用于展示周末或节假日，不作为导入依据
- 合并单元格仅允许出现在标题区
- 排班矩阵区不允许出现公式异常值、图片、批注替代编码等情况
- 排班矩阵区应为完整月表，不允许中间缺列
- `A3` 必须为“姓名”
- 从 `A4` 开始，若某行姓名为空且整行无编码，则视为结束或空行
- 标准模板不解析字体颜色、背景色、加粗、斜体、边框样式

示意结构：

```text
第1行：项目排班表(2026年4月)
第2行：    1   2   3   4   ... 30
第3行：姓名 三   四   五   六   ... 四
第4行：张三 休  捕1 休  休  ... 检2
第5行：李四 休  捕1 休  休  ... 检2
...
```

***

### 8.5 导入映射与校验规则

导入时必须由前端或后台界面明确传入：

- `project_id`
- `schedule_month`

系统处理规则如下：

#### 8.5.1 标题与月份处理

- Excel 标题仅作为展示和辅助校验依据
- 系统以导入参数中的 `project_id` 和 `schedule_month` 为准
- 若标题中的月份与导入参数不一致，系统应报错或强提示
- 标题中的项目名称不作为主匹配依据，不直接解析生成 `project_id`

#### 8.5.2 日期与星期校验

- 第2行日期必须与 `schedule_month` 对应
- 日期应连续递增，从 1 到当月最后一天
- 第3行星期必须与实际日历一致
- 若日期列数、日期内容或星期不匹配，则导入失败

#### 8.5.3 员工匹配规则

- 从 `A4` 开始逐行读取员工姓名
- 系统根据员工姓名匹配员工主数据
- 员工必须存在且处于可排班状态
- 若同名员工存在多条记录，必须报错，不允许自动猜测
- 若员工不存在、已停用或状态不可排班，则报错

说明：

- 本期标准模板按“姓名”导入
- 因模板不含工号列，故要求员工姓名必须能唯一匹配
- 若业务后续存在较多重名场景，应升级为增强版模板，不在本期范围内

#### 8.5.4 单元格值映射规则

每个有效排班单元格映射为一条排班记录：

- `project_id` = 导入参数中的项目
- `schedule_date` = `schedule_month + 列日期`
- `employee_id` = A列匹配到的员工
- `schedule_code` = 单元格编码
- `shift_type` = 根据 `schedule_code` 映射获得
- `planned_hours` = 根据 `schedule_code.standard_hours` 获得
- `task_id` = 空
- `device_id` = 空
- `source_type` = `excel`
- `import_batch_id` = 当前导入批次ID

说明：

- 标准模板不直接导入任务、设备、工时和班次文本
- 计划工时与班次均由编码映射获得

#### 8.5.5 编码校验规则

- 单元格值必须存在于 `schedule_code` 字典中
- 编码必须启用
- 编码必须配置好：
  - 分类
  - 是否计工时
  - 默认工时
  - 映射班次
- 若编码未配置映射规则，则导入失败

#### 8.5.6 空白单元格规则

- 标准月度矩阵模板中，排班矩阵区单元格**不得为空**
- 若为空，视为该员工该日期排班未填写，导入失败
- 若业务确需允许空白，需通过后续版本扩展，不在本期范围内

#### 8.5.7 作业类编码规则

- 对于 `捕1 / 捕2 / 检1 / 检2` 这类作业编码
- 标准模板允许仅导入编码，不强制要求任务和设备
- 导入后生成基础作业班表
- 后续在系统中补录任务和设备后，系统再执行设备技能校验并补齐技能快照

#### 8.5.8 冲突校验

- 同一员工同一日期不得生成多条主排班记录
- 若目标草稿版本中已存在同员工同日期数据，需按导入模式处理：
  - 覆盖当前草稿版本：先删除草稿版本原数据，再导入新数据
  - 新建草稿版本：生成新版本并导入
- 已发布版本不得被直接覆盖

#### 8.5.9 其他格式处理规则

- 单元格值前后空格在导入前自动去除
- 编码匹配时应支持字典配置的别名 `aliases`
- 颜色、公式样式、边框、批注不参与解析
- 若单元格公式结果为空或异常，按导入失败处理

#### 8.5.10 错误定位要求

- 导入失败时，错误信息需精确到单元格
- 错误示例：
  - `A12 员工不存在`
  - `C8 编码不存在`
  - `M4 日期与月份不一致`
  - `A16 员工姓名匹配到多条记录`
  - `R10 单元格为空，不允许导入`

***

### 12.3 关键模型约束

- 场景用于记录名称、地点、说明及设备资源
- 设备必须归属于一个场景
- 项目必须关联一个场景
- 任务必须归属于一个项目
- 任务绑定的设备必须来自项目关联场景
- 设备必须绑定一个技能
- `employee_skill.skill_level` 仅允许取值 `1 / 2 / 3`
- 排班选择设备时，员工必须持有该设备绑定技能
- 排班选择设备时，设备必须已绑定到当前任务
- 标准 Excel 基础导入允许 `task_id` 和 `device_id` 为空
- 排班必须归属于某个排班版本
- 员工端只读取已发布版本
- 导入批次必须可追溯到项目、月份、文件与结果

建议唯一约束：

```text
UNIQUE(mobile, channel_id)
UNIQUE(device_code)
UNIQUE(task_id, device_id)
UNIQUE(employee_id, skill_id)
UNIQUE(version_id, employee_id, schedule_date)
```

建议检查约束：

```text
CHECK (skill_level IN (1, 2, 3))
CHECK (source_type IN ('manual', 'template', 'excel'))
```

## 十四、验收标准

### 14.1 功能验收

- 场景 → 设备 → 项目 → 任务 → 任务绑定设备 → 排班 → 发布 → 查看 → 调班 → 统计闭环可跑通
- 支持多渠道员工管理
- 支持技能级别与效率系数管理
- 支持一个项目关联一个场景
- 支持一个场景维护多个设备
- 支持一个任务绑定多个设备
- 支持一个设备绑定一个技能
- 支持按设备技能校验员工是否可操作设备
- 支持标准 Excel 月排班矩阵导入导出
- 支持调班申请与审批

***

### 14.2 Excel 模板验收

- 支持读取第一个 Sheet
- 支持识别标准模板：
  - 第1行为标题
  - 第2行为日期
  - 第3行为星期，`A3=姓名`
  - `A4` 起为员工姓名
  - `B4` 起为排班编码矩阵
- 支持按自然月识别 28/29/30/31 天
- 支持 `休 / 捕1 / 捕2 / 检1 / 检2 / 事 / 病 / 旷 / 离` 等编码导入
- 单元格颜色不影响导入结果
- 空白排班单元格会报错
- 导入错误可精确定位到单元格
- 同名员工无法唯一匹配时导入失败
- 已发布版本不可直接被导入覆盖

***

### 14.3 数据验收

- 主数据按原则独立建表
- 分类项按原则字典化
- 场景明确为地点与设备资源实体，不作为业务分类字典
- 不包含工位、工种相关模型
- **技能级别不单独建表，存储于** **`employee_skill.skill_level`**
- Excel 基础导入记录允许任务、设备为空
- 排班版本、排班记录、调班申请、导入批次可追溯
- 员工近 7 日、近 30 日工时统计正确

***

### 14.4 权限验收

- 员工无法查看他人数据
- 部门负责人不能越权查看跨部门数据
- 管理员可查看全局数据

***

### 14.5 规则验收

- 设备必须归属场景
- 项目必须关联场景
- 任务绑定设备必须来自项目场景
- 排班选择设备时必须校验员工技能
- 同一设备同一日期同一班次不能重复排班
- 同一员工同一日期在同一版本中不能重复排班
- Excel 导入时排班编码必须可映射到工时与班次
- Excel 导入时姓名必须能唯一匹配员工
